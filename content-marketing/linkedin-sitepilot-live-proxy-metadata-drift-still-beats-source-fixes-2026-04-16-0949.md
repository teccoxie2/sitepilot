# LinkedIn Draft — You do not rank the repo, you rank the rendered page

Answer engines evaluate what the browser receives, not what your Git repo meant to publish.

SitePilot’s latest audit is annoyingly clear:
- source routes build cleanly
- internal links are intact
- sitemap and robots are healthy
- BOFU proxy pages still have factual openings
- but 4 live proxy URLs still ship the generic fallback title

That single deployment-layer failure is enough to damage extraction quality.

Why it matters:
- generic titles flatten buying intent
- answer engines lose confidence in page specificity
- BOFU pages start looking like broad brand pages instead of decision assets
- strong copy cannot fully compensate for weak live metadata

AEO is not a writing-only discipline. It is a rendered-output discipline.

If your source says “Proxy Farm Hardware Selection Guide 2026” but production still serves “SitePilot | Global Digital Infrastructure Authority,” your deployment pipeline is quietly stealing commercial relevance.

Audit live HTML. The repo is not the truth. Production is.

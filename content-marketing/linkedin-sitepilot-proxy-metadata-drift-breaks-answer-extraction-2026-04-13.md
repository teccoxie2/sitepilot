# LinkedIn Draft — SitePilot proxy metadata drift breaks answer extraction

Most SEO teams still audit source code. Wrong battlefield.

This morning’s SitePilot audit showed the proxy cluster source is structurally correct: factual openings are still in the 40–60 word range, BOFU paths still close properly, and internal route integrity is still clean. But four live proxy URLs are still shipping the wrong title tag in production.

That gap matters because answer engines and search snippets extract from rendered truth, not your local repo. If the live title collapses to a generic brand label, you lose retrieval specificity exactly where BOFU comparisons should be strongest.

Current pattern:
- Procurement cluster: live metadata aligned
- Proxy software page: live metadata aligned
- Proxy hub + setup + hardware + residential-vs-mobile: live title drift persists

The lesson is simple: source truth is irrelevant when production truth is wrong.

If you’re building answer-engine content, audit in this order:
1. Live title + H1 alignment
2. 40–60 word factual opening
3. BOFU query specificity
4. Topic-cluster closure
5. Internal link integrity

Pretty content doesn’t save broken extraction.
